## 3. REQUISITOS DO SISTEMA

### 3.1 Requisitos Funcionais

#### RF01 – Cadastro de Usuários
O sistema deve permitir o cadastro de profissionais e empresas, coletando informações básicas como nome, e-mail, senha e tipo de perfil.

#### RF02 – Autenticação de Usuários
O sistema deve permitir login e logout seguro para acesso às funcionalidades da plataforma.

#### RF03 – Criação e Edição de Perfil Profissional
Profissionais devem poder criar e editar seus perfis com informações como habilidades, experiências, formação e objetivos de carreira.

#### RF04 – Criação e Edição de Perfil Empresarial
Empresas devem poder cadastrar e editar informações institucionais, cultura organizacional e dados de contato.

#### RF05 – Publicação de Vagas
Empresas devem poder criar, editar e excluir vagas com descrição, requisitos e benefícios.

#### RF06 – Sistema de Match
O sistema deve permitir que profissionais e empresas curtam perfis/vagas, gerando um “match” quando houver interesse mútuo.

#### RF07 – Exibição de Recomendações
O sistema deve sugerir vagas para profissionais e candidatos para empresas com base em compatibilidade de perfil.

#### RF08 – Sistema de Notificações
O sistema deve notificar usuários sobre novos matches, mensagens ou atualizações relevantes.

#### RF09 – Chat entre Usuários
Após um match, o sistema deve permitir comunicação direta entre empresa e candidato.

#### RF10 – Filtro e Busca
O sistema deve permitir que usuários filtrem vagas ou candidatos por critérios como localização, área, habilidades e nível de experiência.

---

### 3.2 Requisitos Não Funcionais

#### RNF01 – Portabilidade
O sistema deve poder ser acessado via navegador sem necessidade de instalação obrigatória.  
Justificativa: Facilita o uso e reduz barreiras de entrada para novos usuários.

#### RNF02 – Desempenho
O sistema deve responder às ações do usuário em até 3 segundos em condições normais de uso.  
Justificativa: Garante boa experiência e evita abandono da plataforma.

#### RNF03 – Usabilidade
A interface deve ser intuitiva, responsiva e de fácil navegação, inspirada em aplicativos de relacionamento.  
Justificativa: O diferencial do projeto é a experiência simples e engajante.

#### RNF04 – Escalabilidade
O sistema deve ser capaz de suportar aumento de usuários sem perda significativa de desempenho.  
Justificativa: Essencial para crescimento da plataforma SaaS.

#### RNF05 – Disponibilidade
O sistema deve estar disponível 99% do tempo, com mínima indisponibilidade.  
Justificativa: Usuários e empresas dependem do acesso contínuo para recrutamento.
